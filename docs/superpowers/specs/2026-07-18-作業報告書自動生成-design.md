# 作業報告書 自動生成システム 設計書

作成日: 2026-07-18 / 著者: 安藤至大

## 目的

Claude Code での前日の作業を、MacBook のスリープ復帰をトリガーとして自動でまとめ、単一の累積 HTML ファイル（作業報告書）に日ごとの記録として積み上げていく。1 年分の振り返りができるよう、最新の記録を上に、各月へのナビゲーションを備える。「一つの通しファイルに、取り組みが地層のように積み上がる」ことに意味を置く。

## 決定事項（ブレインストーミングで確定）

- **要約方式**: Claude による要約（`claude -p` ヘッドレス呼び出し）で自然な報告書を生成する。
- **生成タイミング**: 1 日 1 回。その日最初のスリープ復帰時に「前回記録〜現在」をまとめる。同日 2 回目以降の復帰では何もしない。
- **各日の記録内容**: ①作業内容の要約文 ②主なトピックの箇条書き ③軽い統計指標（作業時間帯・ターン数など）。ファイル一覧は含めない。
- **復帰検知**: `sleepwatcher`（Homebrew）の wake フックから起動する。
- **対象範囲**: `~/.claude/projects` 配下の全プロジェクト。ただし本システム自身（kiroku）は除外。
- **データ方針**: `entries.json`（全記録の元データ）を真実の源とし、そこから毎回 `作業報告書.html` を全再生成する。単一累積 HTML という要件は結果として満たす。
- **初回実行**: 過去 1 ヶ月を対象とし、**日ごとにバックフィル**して各日のエントリを生成する。

## 全体構成

```
[sleepwatcher wake フック] --> run-kiroku.sh
                                   │
    ┌──────────────────────────────┼──────────────────────────────┐
    ▼                              ▼                               ▼
 ① gather.py                  ② claude -p 要約               ③ render.py
 前回記録以降の jsonl を       ①のダイジェストを読み          entries.json から
 プロジェクト別に抽出し        プロジェクトごとの要約文＋      作業報告書.html を
 ダイジェスト化               箇条書きを JSON で生成          毎回まるごと再生成
    │                              │                               ▲
    └──────> entries.json に当日（または各日）エントリを追記 ──────┘
```

## 各コンポーネント

### ① gather.py（決定的・LLM なし）

- **入力**: `state.json` の `last_recorded_ts`（前回記録タイムスタンプ）と `last_recorded_date`。
- **1 日 1 回ガード**: `last_recorded_date` が今日なら即終了（exit 0、何もしない）。
- **対象ディレクトリ**: `~/.claude/projects/*`。ただし kiroku 自身（`-Users-munetomoando-claude-work-kiroku`）は除外。
- **抽出**: 各 jsonl を走査し、`last_recorded_ts` 以降のメッセージを日付（ローカル）× プロジェクト（cwd 由来）で束ねる。各束から以下を抽出:
  - ユーザー指示（`type=user` のテキスト）
  - アシスタントの主な動き（要約に足るテキスト。ツール多用箇所は圧縮）
  - 統計: 最初/最後のタイムスタンプ（作業時間帯）、ユーザーターン数、アシスタントターン数
- **出力**: 要約用ダイジェスト（日付 × プロジェクト構造のテキスト/JSON）を stdout もしくは一時ファイルへ。
- **初回（state.json なし）**: 対象開始を「今日の 1 ヶ月前」に設定し、日ごとに束ねてダイジェスト化（各日が独立エントリになる）。

### ② claude -p（要約）

- gather のダイジェストを渡し、**プロジェクトごとに**「要約文（数行）＋主なトピックの箇条書き」を日本語で生成。
- 出力は構造化 JSON（`{日付: {プロジェクト名: {summary, bullets}}}`）で受け取り、パース失敗に備える。
- 複数日バックフィル時は日ごとに要約（1 プロンプトに複数日を渡し日別 JSON を返させる、もしくは日ごとに呼び分け。実装計画で確定）。

### ③ render.py（決定的）

- ②の要約結果に gather の統計を合成し、当日（または各日）のエントリを `entries.json` に追記。
- `entries.json` 全体から `作業報告書.html` を全再生成:
  - 冒頭: タイトル＋説明、月ナビ（`2026年7月` `2026年6月` … 各月ボタン）。
  - 月ナビのアンカー: その月の**最も古い（一番下の）エントリ**に張る。
  - 本体: 日付セクション（**最新が上**）→ プロジェクト見出し（h3）→ 要約文・箇条書き・軽い統計。
  - 著者表記: 安藤至大。
- 生成後、`state.json` を更新（`last_recorded_ts` = 今回処理の最終タイムスタンプ、`last_recorded_date` = 今日）。

### run-kiroku.sh（オーケストレーション）

1. `gather.py` 実行 → 記録すべき作業が無ければ終了。
2. ダイジェストを `claude -p` に渡し要約 JSON を取得。
3. `render.py` で entries.json 追記＋HTML 再生成＋state.json 更新。
4. ログを残す（`kiroku.log`）。

## ファイル配置（すべて `kiroku/` 内）

| ファイル | 役割 |
|---|---|
| `作業報告書.html` | 成果物（累積・単一ファイル） |
| `entries.json` | 全記録の元データ（真実の源） |
| `state.json` | 前回記録タイムスタンプ・最終記録日 |
| `gather.py` | 抽出（決定的） |
| `render.py` | 追記＋HTML 再生成（決定的） |
| `run-kiroku.sh` | オーケストレーション |
| `kiroku.log` | 実行ログ |

wake フック: `~/.sleepwatcher`（の wakeup スクリプト）から `run-kiroku.sh` を呼ぶ。

## エラーハンドリング

- **要約失敗（②）**: 機械抽出の箇条書き（①由来）だけで最低限のエントリを残し、記録の欠落を防ぐ。
- **HTML 生成失敗（③）**: entries.json は追記済みなので、次回実行時に再生成でリカバリ可能。書き込みは一時ファイル → アトミック置換。
- **jsonl パース不正行**: 行単位でスキップし継続。
- **多重起動**: ロックファイルで排他（同一 wake で複数起動しても 1 回に収束）。

## テスト方針

- `gather.py`: サンプル jsonl を入力に、日付 × プロジェクトの束ね・統計・1 日 1 回ガード・kiroku 除外を検証。
- `render.py`: サンプル `entries.json` から HTML を生成し、最新が上・月ナビのアンカー・プロジェクト見出し構造を検証。
- 初回バックフィル: state.json なしで過去 1 ヶ月が日ごとに分割されることを検証。
- 統合: `claude -p` はモック（固定 JSON）に差し替えて run-kiroku.sh の一連の流れを検証。

## スコープ外（YAGNI）

- リアルタイム/復帰ごとの追記。
- Web ホスティングや共有機能（ローカルの単一 HTML を開くのみ）。
